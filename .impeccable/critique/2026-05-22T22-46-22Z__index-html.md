## Form validation upgrades

- Inline blur validation em nome / email / whatsapp (catches errors antes do submit)
- aria-invalid="true" + aria-describedby ativados dinamicamente para SR users
- Smart constraints: whatsapp aceita 10-15 dígitos (sem máscara obrigatória, aceita formato livre)
- inputmode="tel" no whatsapp (mobile keyboard correto)
- Mensagens específicas com exemplo de recovery: "Formato de e-mail inválido (ex: voce@empresa.com)"
- Forgiving UX: erro limpa automaticamente ao usuário começar a digitar (não fica enchendo o saco)
- Loading state visual: spinner CSS animado + label "Enviando" durante fetch
- aria-live="polite" no painel de erro de rede (anunciado por SR sem interromper)
- Network failure handling: button vira "Tentar de novo →" + painel vermelho com diagnóstico + mailto link com subject pré-formatado conforme serviço selecionado
- Dados do form preservados em caso de erro (não force re-fill)

## Service pills (Miller's Law fix)

- 5 pills lado-a-lado → 4 single services + separador "ou" + Pacote Completo destacado com tinge magenta + border violeta-fuchsia
- Bundle conceptualmente separado do menu de serviços individuais
- Visual hierarchy: TIKTOK SHOP / ECOMMERCE / DEV GENERATIVO / AUTOMAÇÃO em uma faixa; PACOTE COMPLETO como alternativa visualmente diferenciada
